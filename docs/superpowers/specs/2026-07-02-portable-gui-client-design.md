# SS-RS 便携 GUI 客户端设计

## 目标

为 Windows x64 提供可分享的便携 GUI。用户解压 ZIP 后双击 `SS-RS.exe`，通过单个按钮启动或停止全局 TUN 隧道，不需要安装 Rust 或手动配置代理。

## 发布形式

ZIP 只包含：

- `SS-RS.exe`：egui 图形入口，自带管理员权限清单。
- `ss-client.exe`：现有代理与 TUN 引擎。
- `wintun.dll`：官方签名的 x64 Wintun 驱动库。

服务器 `23.27.52.126:8388` 与密码在打包时嵌入 `SS-RS.exe`。密码不出现在命令行，但可从分发程序中提取；分享客户端等同于分享该账号。

## 界面

采用极简素描风：纸张底色、炭笔文字、单色细线、无渐变，只有连接成功状态使用低饱和绿色。单窗口显示：

- SS / RS 标识和连接状态；
- Wintun、服务器、全局 TUN 模式；
- 最近 5 行运行日志；
- 启动隧道、断开连接或重试按钮；
- 关闭窗口会安全断开并恢复网络的提示。

不提供设置页、托盘、自动重连、开机启动或多服务器管理。

## 运行架构

新增独立 GUI crate，使用 egui/eframe。GUI 以隐藏窗口方式启动同目录的 `ss-client.exe --tun`，通过 `SS_PASSWORD` 环境变量传递嵌入密码，并捕获 stdout/stderr。

引擎新增 GUI 子进程模式：标准输入关闭或收到停止命令时走现有正常退出路径，确保删除全局路由、恢复 DNS 并关闭 Wintun。GUI 在关闭前先发出停止命令并等待进程退出；超时才强制终止。

状态机固定为：未连接 → 连接中 → 已连接 → 正在断开 → 未连接；子进程意外退出进入连接失败。读到 `TUN active` 日志后进入已连接。错误状态保留最后日志，并将按钮改为重试。

缺失 `ss-client.exe`、`wintun.dll`、管理员权限、服务端不可达或认证失败时，窗口显示中文错误，不退出 GUI。

## 打包

PowerShell 打包脚本从本机 DPAPI 密码文件读取密码，通过构建环境变量嵌入 GUI，构建 release 后复制三个发布文件并生成 ZIP。脚本拒绝缺少密码、无效 Wintun 签名或构建失败的产物。

## 验收

- GUI 可启动、连接、断开和重试；关闭后无残留进程、路由或 DNS 修改。
- 缺失引擎/DLL、密码错误及服务端离线均显示明确中文错误。
- 连接后 TCP、UDP DNS、1MB 下载通过，出口 IP 为服务器。
- ZIP 在未安装 Rust 的 Windows x64 电脑上解压即用，仅首次启动弹出 UAC。
- 自动测试覆盖子进程状态转换、最近 5 行日志、正常停止和异常退出；现有协议测试继续通过。
