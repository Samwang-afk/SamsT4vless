# SS-RS 响应式界面与按钮反馈设计

## 目标

修复客户端窗口不可缩放、内容不能随窗口宽高适配，以及连接按钮缺少明确悬停和按下反馈的问题。保持现有极简素描风，不改变代理、流量统计和服务端逻辑。

## 窗口与布局

- 窗口允许自由缩放，初始尺寸保持 `430 × 410`，最小尺寸保持 `390 × 390`。
- 字体大小不随窗口缩放。
- 页面外边距根据可用宽度在 `16–28px` 内变化。
- 流量卡片始终等宽平分可用宽度，不使用固定最小宽度。
- 日志区使用剩余垂直空间，并设置最小高度；窗口变高时日志区优先增长。
- 连接按钮始终填满可用宽度；窄窗口不得出现横向溢出。

## 按钮反馈

- 默认态：纸色按钮、2px 炭笔边框，右下方绘制 5–8 条 135° 斜线形成偏移阴影。
- 悬停态：按钮背景略深，斜线颜色加深、阴影偏移略增。
- 按下态：按钮向右下移动约 3px，深色填充、浅色文字，斜线阴影面积收缩。
- 禁用态：降低文字、边框和斜线对比度，不显示按压反馈。
- 使用 egui painter 直接绘制斜线，不新增图片、字体或 UI 依赖。

## 验收

- 窗口可从最小尺寸拖动放大，所有元素随可用空间重新布局。
- 最小尺寸下流量卡片、日志和按钮均不被裁切且无横向溢出。
- 鼠标悬停、按下、释放按钮时能清楚看到三种状态变化。
- 连接、断开、忙碌禁用状态和现有 GUI 单元测试保持正常。
