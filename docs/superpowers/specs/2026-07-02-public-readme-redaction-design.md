# 公开 README 与仓库脱敏设计

## 目标

让同学能够理解、构建和学习 SamsT4vless，同时不公开个人服务器地址、域名、面板端口、账号或部署记录。

## README

README 使用简洁中文，包含：

- 项目定位与学习用途声明；
- SOCKS5、HTTP、Windows TUN、加密隧道和 VLESS 客户端等主要能力；
- Rust workspace 目录说明；
- 不含真实节点信息的构建与运行示例；
- 安全提示、当前限制和贡献方式。

所有示例仅使用 `example.com`、`SERVER_ADDRESS` 等占位符，不提供可直接连接的公共节点。

## 脱敏范围

- 将 GUI 中的固定服务器地址改为构建时环境变量，无值时拒绝生成可连接配置。
- 将启动脚本中的固定服务器地址改为调用方提供的参数或环境变量。
- 将部署脚本中的个人面板端口改为显式配置项。
- 从公开仓库移除包含个人部署信息的项目提示和旧设计规格；文件保留在本地并加入忽略规则。
- 不提交 EXE、构建目录、本地密码文件、证书、私钥或面板配置。

## Git 历史

因为真实基础设施信息已存在于公开提交，最终公开分支将重建为一个干净的根提交并强制推送。仓库 README 和项目源码保留，旧公开提交不再由分支或标签引用。

历史重写不能保证已被第三方缓存或克隆的数据消失，因此相关地址不能继续被视为秘密；凭据仍应独立轮换。

## 验证

1. 全仓扫描真实 IP、域名、面板端口、密码、Token 和私钥标记。
2. 运行 `cargo test --workspace`。
3. 验证默认构建不含可连接节点，带构建参数时配置能够生成。
4. 强制推送后重新克隆远程仓库，并对全部可达提交执行同样的扫描。

## 不包含

- 不发布现成节点、客户端凭据或个人部署教程。
- 不删除本地内部文档。
- 不借此次脱敏重构无关代码或改变代理协议。
